视频内容：  

目录的地方详细说说

说明一下 本篇视频的主要内容是如何构建chatgpt的输入， 为了缩短视频的时长，我们将会简单的看一下生成出现的代码， 但是不会深究。 

写一个工具将软件1的配置文件转换成软件2的。 
构建输入的模式是 转换示例 + 额外要求。 

函数内容填充， 一些工具函数， 或者小型函数内容的填充。 
构建输入的模式是 提供一些相关联的上下文， 然后使用意义良好的函数名和参数名

UI 代码生成   
模式是： 提供一个结构， 提供框架的名称， 提供一些附加需求。 

finish to do for me  函数写了一半， 剩下的一半都是很简单的逻辑。
现在有点累了， 怎么办呢。 
就可以写点注释作为引导， 然后将函数内容丢给chatgpt
模式是 前置信息 + 现在的代码块

数学问题， 数学基础差的话， 解决数学问题需要的时间久， 解决起来令人头大。 
所以可选的方案是整理现在存在的信息， 想要求解的信息， 合并起来发送过去。 

linux / windows 脚本。   边查文档边写边调试， 程序员应该都可以实现需求。  但是这种情况下， 效率极差。
chat gpt 可以1分钟完成你几个小时的工作，这对效率的提升是多么巨大。 
这个的模式很简单， 就是添加 linux 脚本， 或者 windows cmd/powershell 作为关键词，然后添加上自己的需求就可以了。 

和使用搜索引擎的对比，   使用搜索引擎需要搜索相关联的内容， 然后转换到当前场景里面的内容。 你需要参考别人的方案，然后自己写内容。   chatgpt将帮你跳过转换的过程， 直接帮你转换好了， 你不需要自己再次进行转换。  
所以不要像使用搜索引擎那样来使用chatgpt， 提供给他们更多的信息。

翻译是常用操作。  

翻译文章的同时， 可以让它直接帮我们提取要点， 这样就可以节省阅读大段文字的时间。

chat with doc 

将pdf 文档按照段落或者其他方式切块， 将切出来的块转换成向量， 然后保存到向量数据库内。
将用户的输入转换成向量， 从向量数据库中查询相似性高的几个段落。 
随后将用户的输入和查询出来的段落使用一个特定的模板组成一个需求， 发送给chatgpt
所以实际上， chat gpt 并不是阅读了这本书， 然后再回答你内容的。 

open-interpreter 在本地使用自然语言处理一些文件，  感觉有点用， 但是不多。 尤其是网络不好的时候， 很容易产生很多花费。

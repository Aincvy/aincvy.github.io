# 0. K8s的简要教程 - 开篇


使用k8s， 我们大致可以获得下面的好处。 
- 自动化部署
- 动态扩缩
- 灾难恢复
- 回滚版本
- 管理方便
- 负载均衡

有好处， 自然有缺点。 下面是一些缺点。
- 国内安装比较困难， 大多都是因为网络的问题
- k8s自身需要占用一些资源。 
- 基于容器， 容器化的底层还需要占用一些资源

对于大型项目来说， 这些缺点都是一些无足轻重的小问题。 而对于小型项目来说， 还是手动写更新脚本比较划算一点。

相比于游戏项目来说， 网站项目好像是更适合一些， 微服务更适合。


这个小系列将会包含下面的内容
- k8s的简单使用
- k8s的安装
- 服务导出以及动态卷挂载
